Flag Reference Definitions
==========================

Top Level Flags
---------------
is_relevant: true | false

If false, the Backend ignores all other fields and returns direct_response to the user.
If true, the Backend proceeds to execute the RAG logic.

direct_response: string | null
Used only when is_relevant: false.

Example: "I am designed to assist with clinical analysis. Please ask a question related to the therapy sessions."

A. rag_strategy.intent (What is the goal?)
------------------------------------------

retrieval: Find specific information or moments.
comparison: Compare two data points (requires 2 distinct searches/filters).
summarization: Compress large amounts of text into a summary.
aggregation: Count occurrences (e.g., "How many times...").

B. rag_strategy.execution_mode (How do we search?)
--------------------------------------------------

hybrid_search: (Default) Vector Similarity + Keyword Filter (Qdrant).
full_fetch_as_is: (No Vectors) SQL SELECT * from DB for the whole session.
keyword_only: (No Vectors) SQL LIKE or Full-Text Search (Exact matches).
count_only: (No Text) SQL SELECT COUNT(*) (Fast stats).

C. rag_strategy.data_level (What table do we look at?)
------------------------------------------------------

utterance: Search individual dialogue lines (Granular).
session_summary: Search only high-level session summaries (Broad).
session_metadata: Search only dates, durations, or tags (Administrative).

D. rag_strategy.response_format (What does the Frontend need?)
--------------------------------------------------------------

text: Standard string response.
audio_ref: Returns text + Audio URL + Start/End Timestamps (for Player).
list: Returns a structured array (e.g., list of medications).
number: Returns a raw integer (e.g., "14 times").

E. rag_strategy.relational_logic (Do we hop to another row?)
------------------------------------------------------------

direct_match: (Default) Return the exact chunk found by search.
next_utterance: Find Match X -> Return Sequence_ID(X) + 1 (Reaction).
previous_utterance: Find Match X -> Return Sequence_ID(X) - 1 (Context).

F. scope.mode (Which sessions?)
-------------------------------

all_sessions: Search across every session for this client.
latest_session: Find MAX(session_number) and restrict search to it.
specific_sessions: Restrict search to target_session_numbers (e.g., [1, 5]).
range: Restrict search to sessions between X and Y.
last_n_sessions: Sort descending and take top N sessions.

G. filters (Constraints)
------------------------

search_target_speaker: Who to search in vectors? (Client or Therapist).
return_source_speaker: Who to return to user? (Client or Therapist).
clinical_themes: Filter by AI-detected concepts (e.g., "Trauma").
keywords: Filter by exact text tokens (e.g., "Xanax").

2. Flag Reference & Definitions
Use this guide to implement the logic in rag_service.py and vector_service.py.

A. rag_strategy.intent (The Goal)
retrieval: Find specific moments or quotes.

Query: "What did he say about his mother?"

comparison: Compare two different data points or sessions.

Query: "How is his anxiety today compared to Session 1?"

summarization: Compress large text blocks into a summary.

Query: "Give me a recap of the last 3 sessions."

aggregation: Calculate statistics or time metrics.

Query: "How many times did he mention suicide?" or "How long did we talk about trauma?"

B. rag_strategy.execution_mode (The Search Method)
hybrid_search: (Default) vector_similarity + keyword_filter (Qdrant). Best for semantic meaning.

full_fetch_as_is: (No Vectors) SQL SELECT *. Used when the user wants "everything" from a session.

keyword_only: (No Vectors) SQL ILIKE. Used for exact names or medication lookup.

count_only: (No Text) SQL COUNT(*). Used for fast statistics.

C. rag_strategy.response_format (The Frontend Output)
text: Standard string response.

audio_ref: Returns text + Audio URL + Start/End Timestamps (Triggers Audio Player).

list: Returns a JSON array (e.g., bullet points of topics).

number: Returns a raw integer or float (e.g., "14" or "12.5 minutes").

D. rag_strategy.relational_logic (The "Hop" Logic)
direct_match: (Default) Return exactly what was found in the search.

next_utterance: Search X -> Return Sequence_ID(X) + 1. (Used for "What was his reaction?").

previous_utterance: Search X -> Return Sequence_ID(X) - 1. (Used for "What led to that?").

transition_search: Search Topic A and Topic B -> Return the chunk where they appear within N seconds of each other. (Used for "When did we switch topics?").

E. rag_strategy.aggregation_type (Math Logic)
null: (Default) No math required.

count: Count the number of rows found.

sum_duration: Calculate SUM(end_seconds - start_seconds) for all matches.

F. scope.mode (Session Filtering)
all_sessions: Search the entire history.

latest_session: Restrict search to MAX(session_number).

specific_sessions: Restrict to target_session_numbers (e.g., [1, 5]).

range: Restrict to sessions between X and Y.

last_n_sessions: Sort descending and take top N.

G. filters.filter_mode (Negative Logic)
include: (Default) Standard search. Find where the keywords exist.

exclude: Find sessions/utterances where the keywords do NOT exist.

Query: "Show me sessions where he did NOT mention his wife."

H. filters (Extra Parameters)
search_target_speaker: Who to search? (Client or Therapist).

return_source_speaker: Who to show? (Client or Therapist).

start_topic / end_topic: Used only when relational_logic is "transition_search".
